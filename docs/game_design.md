## 1. 核心概念

### 1.1 游戏主题

你是一名都市调查记者，带着相机深入"暗面都市"的各个角落。白天按日程行动、搜集线索；夜晚翻开隐藏的事件卡牌，面对怪物、陷阱、奖励与阴谋。你的目标是在 **3 天内存活**，用镜头记录真相，用光明驱散恐惧。

**核心口号**: "用镜头记录真相 · 用光明驱散恐惧"

### 1.2 核心玩法循环

```
新的一天开始
  → 日期转场动画 (Persona 4 风格)
  → 棋盘发牌 (5×5 螺旋发牌)
  → 玩家在棋盘上移动 (点击相邻卡牌)
  → 翻开卡牌 → 触发事件 (弹窗)
  → 结算事件效果 (资源增减)
  → 可随时:
      - 使用相机 (远程拍摄/驱魔)
      - 打开手账本 (查看日程/传闻/道具)
      - 进入商店 (购买道具)
  → 点击 "结束这一天"
  → 日程结算 (奖惩)
  → 检查胜负条件
  → 进入下一天 / 游戏结束
```

### 1.3 胜负条件

| 条件 | 结果 |
|------|------|
| 存活 MAX_DAYS(3) 天 | **胜利** - "任务完成" |
| 理智 (San) ≤ 0 | **失败** - "意识崩溃" |
| 秩序 (Order) ≤ 0 | **失败** - "意识崩溃" |

## 2. 资源系统

### 2.1 四种资源

| 资源 | 英文 Key | 图标 | 初始值 | 最大值 | 说明 |
|------|----------|------|--------|--------|------|
| 理智 | `san` | 🧠 | 10 | 10 | 精神健康，被怪物/陷阱扣减 |
| 秩序 | `order` | ⚖️ | 10 | 10 | 社会秩序感，被怪物/陷阱扣减 |
| 钱币 | `money` | 💰 | 50 | ∞ | 经济资源，用于商店购物 |
| 胶卷 | `film` | 📷 | 3 | ∞ | 相机弹药，拍摄/驱魔消耗 |

### 2.2 资源变化规则

- `san` 和 `order` 有上限 (10)，不能超出
- `money` 和 `film` 无上限
- 任意资源降到 0 以下时 `clamp` 到 0
- `san ≤ 0` 或 `order ≤ 0` 触发游戏失败

## 3. 棋盘与卡牌

### 3.1 棋盘规格

- **尺寸**: 5 行 × 5 列 = 25 张卡
- **间隔**: GAP = 0.12 米 (3D 世界坐标)
- **牌堆位置**: DECK_X = 3.0, DECK_Z = -1.5 (右下方牌堆)
- **发牌顺序**: **螺旋序** (从外圈到内圈顺时针旋转)

### 3.2 卡牌物理尺寸 (3D)

- 宽: CARD_W = 0.64 米
- 高: CARD_H = 0.90 米
- 厚度: CARD_THICKNESS = 0.015 米
- 贴图分辨率: 256 × 360 像素

### 3.3 双层卡牌系统

每张卡牌有两个层:

| 层 | 名称 | 初始状态 | 内容 |
|----|------|----------|------|
| 表层 | 地点层 (Location) | 面朝上可见 | 城市中的地点名称与图标 |
| 里层 | 事件层 (Event) | 面朝下隐藏 | 隐藏事件 (怪物/陷阱/奖励等) |

**翻牌机制**: 玩家移动到一张卡上 → 自动翻转 → 地点面翻到背面 → 事件面朝上 → 触发事件弹窗

### 3.7 地标光环机制

当一张卡是 **地标** (lighthouse/library/temple) 时:
- 检测其上下左右 4 个相邻格子
- 如果相邻格子的事件是 `monster` 或 `trap`
- 将该事件 **转化为 `safe`** (安全)
- 这模拟了"圣地净化周围黑暗"的效果

### 3.8 卡牌动画

| 动画名 | 描述 | 参数 |
|--------|------|------|
| `dealTo` | 从牌堆飞到指定位置 | 4 阶段: 透明度→位置→缩放→高度 |
| `undeal` | 反向收回牌堆 | dealTo 的逆过程 |
| `flip` | 翻牌 | 压缩→抬升(0.36m)→90°旋转→换贴图→落下 |
| `flipBack` | 反向翻回 | flip 的逆过程 |
| `flipToFace` | 相机模式翻到正面 | 简化版 flip |
| `flipToBack` | 相机模式翻回背面 | 简化版 flipBack |
| `shake` | 抖动 | 阻尼正弦波 (衰减振荡) |
| `transformTo` | 怪物→照片变身 | 抖动→抬升→旋转换贴图 |

### 3.9 卡牌生成规则

1. **固定卡位**:
   - 中心 (3,3) = home (家)
   - 随机 1 格 = shop (商店)
   - 3 张 = landmark (地标: lighthouse/library/temple)

2. **日程必选地点**: CardManager 的 `preSelectLocations()` 确保当天日程目标地点出现在棋盘上

3. **事件加权随机**: safe:30, monster:20, trap:15, reward:15, plot:10, clue:10

4. **地标光环处理**: 生成完毕后扫描地标，净化相邻的 monster/trap → safe

## 4. 日程与传闻系统

### 4.1 日程 (Schedule)

每天生成 **3 个日程任务**，每个日程包含:

| 字段 | 说明 |
|------|------|
| `location` | 目标地点 Key |
| `verb` | 动作描述 (如 "拍摄公司照片") |
| `reward` | 完成奖励 (如 `{money = 10}`) |
| `status` | `"pending"` / `"completed"` / `"deferred"` |

#### 日程模板 (12 种)

| 地点 | 动作 | 奖励 |
|------|------|------|
| company | 拍摄公司照片 | money +10 |
| park | 在公园取景 | san +1 |
| hospital | 调查医院档案 | money +8 |
| school | 记录学校传闻 | order +1 |
| station | 追踪车站线索 | money +12 |
| market | 拍摄商场橱窗 | money +8 |
| shrine | 参拜神社 | san +2 |
| alley | 探索巷子深处 | money +15 |
| lighthouse | 登上灯塔瞭望 | order +2, san +1 |
| library | 查阅图书馆文献 | order +1 |
| temple | 在教堂祈祷 | san +2 |
| home | 回家整理笔记 | san +1, order +1 |

#### 日程生成规则

1. 每天随机 3 个日程 (不重复地点)
2. 昨日延期的日程 (deferred) 会追加到今天 (最多 +1)
3. 日程目标地点会通过 `preSelectLocations()` 注入棋盘生成

#### 日程结算

| 状态 | 结算效果 |
|------|----------|
| `completed` | 获得奖励 (显示 VFX 飞字) |
| `deferred` | 延期到下一天 (保留) |
| `pending` (未完成) | 秩序 -1 惩罚 |

### 4.2 传闻 (Rumor)

每天生成 **1 条传闻** (安全提示或危险警告):

| 类型 | 示例 | 来源 |
|------|------|------|
| 安全提示 | "📍 听说 **公园** 附近还算安全" | 随机选取一个 safe 类型卡牌的地点 |
| 危险警告 | "⚠️ 有人说 **医院** 附近有不好的东西" | 随机选取一个 monster/trap 类型卡牌的地点 |

- 每当翻开 `clue` 类型卡牌时，**额外追加一条传闻**
- 传闻以便签纸样式展示在手账本中

### 4.3 玩家交互

- **完成日程**: 玩家移动到日程目标地点并翻开卡牌 → 日程自动标记为 `completed`
- **延期日程**: 在手账本中点击未完成日程 → 切换为 `deferred`；再次点击可取消延期
- **不可延期已完成的**: `completed` 状态的日程不能被点击

## 5. 相机系统

### 5.1 概述

相机是核心特色机制，提供两种能力:
- **拍摄 (Photograph)**: 远程翻开一张未翻的卡牌 (无需移动到旁边)
- **驱魔 (Exorcise)**: 将一张已翻开的怪物卡转化为照片卡 (无害化)

### 5.2 UI 组件

- **相机按钮**: 右下角浮动圆形按钮 (直径 44px)
- **胶卷计数器**: 始终显示 (📷 × 数量)
- **取景框**: 进入相机模式后全屏覆盖
  - 四角 L 形标记线
  - 水平扫描线 (60px/s 匀速下移)
  - REC 指示灯 (闪烁红点 + "REC" 文字)
  - 暗角特效 (四角渐变黑暗)

### 5.3 操作流程

```
点击相机按钮 → 进入相机模式
  → 所有未翻卡牌翻到正面 (显示地点)
  → 所有已翻怪物卡高亮显示
  → 取景框覆盖层出现
  
点击未翻卡牌 → 拍摄
  → 消耗 1 胶卷
  → 远程翻开该卡牌
  → 触发事件弹窗
  → 退出相机模式

点击已翻怪物卡 → 驱魔
  → 消耗 1 胶卷
  → 怪物卡 transformTo → 照片卡
  → 退出相机模式

胶卷不足 → 按钮抖动动画
```

### 5.4 消耗

- 每次拍摄或驱魔消耗 **1 胶卷**
- 胶卷可通过商店购买 (10 钱币) 或奖励事件获取

## 6. 商店系统

### 6.1 概述

玩家移动到 `shop` 卡牌时，自动打开商店弹窗。商店展示 3 件随机商品，以卡片形式排列。

### 6.2 商品列表 (7 种)

| Key | 名称 | 图标 | 价格 | 类型 | 效果 |
|-----|------|------|------|------|------|
| `coffee` | 咖啡 | ☕ | 8 | 消耗品 | 理智 +2 |
| `film` | 胶卷补充 | 🎞️ | 10 | 消耗品 | 胶卷 +1 |
| `shield` | 护身符 | 🛡️ | 15 | 持久品 | (被动效果) |
| `exorcism` | 退魔香 | 🔮 | 12 | 消耗品 | (主动使用) |
| `map` | 都市地图 | 🗺️ | 10 | 消耗品 | (信息获取) |
| `sedative` | 镇定剂 | 💊 | 12 | 消耗品 | 理智 +3 |
| `orderManual` | 秩序手册 | 📘 | 10 | 消耗品 | 秩序 +2 |

### 6.3 商店机制

- **展示**: 从 7 种商品中随机选 3 种 (不重复)
- **刷新**: 花费 **5 钱币** 刷新商品 (重新随机 3 种)
- **购买**: 点击商品卡 → 扣除钱币 → 获得物品 → 卡片消失动画
- **钱不够**: 卡片抖动反馈 (shake 动画)

### 6.4 3 种商店变体

| 变体 | 名称 | 欢迎语 |
|------|------|--------|
| 1 | 午夜便利店 | "深夜好，有什么需要的吗？" |
| 2 | 黄昏杂货铺 | "嘿，看看有什么中意的？" |
| 3 | 巷口自贩机 | "请投币选择商品。" |

### 6.5 道具栏

- **消耗品** (coffee, sedative, orderManual, film, exorcism) 进入随身道具栏
- 道具栏显示在手账本底部的工具条中
- 道具有固定排序: shield → exorcism → coffee → sedative → orderManual
- 每个道具显示图标 + 数量，悬停时弹出 tooltip

### 6.6 卡片动画

| 动画 | 触发 | 效果 |
|------|------|------|
| 入场 | 商店打开 | 卡片交错入场 (stagger)，带旋转 |
| 悬停 | 鼠标移入 | 卡片抬升 + 发光边框 |
| 购买 | 点击购买 | 闪光 → 脉冲 → 消失 |
| 抖动 | 钱不够 | 快速左右晃动 |

## 8. 游戏流程与状态机

### 8.1 两级状态机

#### 第一级: 游戏阶段 (gamePhase)

```
"title" → "playing" → "gameover"
           ↓
         (重新开始 → "title")
```

#### 第二级: 演出状态 (demoState) — 仅在 "playing" 阶段

```
"idle"
  → "dealing" (发牌动画中)
    → "ready" (等待玩家操作)
      ↔ "flipping" (卡牌翻转中)
        → "popup" (事件弹窗展示中)
          → "ready" (回到等待)
      
      → "moving" (Token 移动中)
        → "flipping" → "popup" → "ready"
      
      → "photographing" (拍摄中)
        → "popup" → "ready"
      
      → "exorcising" (驱魔中)
        → "ready"
```

### 8.2 一天的完整流程

```lua
1. advanceDay()
   → currentDay += 1
   → 检查是否超过 MAX_DAYS → checkVictory()
   → DateTransition.play(currentDay)
   → onDateTransitionComplete:
     a. Board.clear() — 清除旧棋盘
     b. CardManager.generateDaily(currentDay) — 生成日程
     c. Board.generateCards() — 生成新棋盘
     d. CardTextures.preloadBoard() — 预加载贴图
     e. Board.startDeal() — 开始螺旋发牌动画
     f. Token.show() — 显示角色在中心
     g. HandPanel.setup() — 初始化手账本

2. 玩家阶段 (demoState = "ready")
   → 点击卡牌 → 移动 + 翻牌 + 事件
   → 使用相机 → 拍摄 / 驱魔
   → 打开手账本 → 查看日程/传闻/道具
   → 打开商店 → 购买道具

3. endDay() — 点击"结束这一天"
   → CardManager.settleDay() — 日程结算
   → 显示结算奖惩动画
   → advanceDay() → 下一天
```

### 8.3 重新开始

```lua
onGameRestart()
  → 重置所有资源到初始值
  → Board.clear()
  → Token.hide()
  → currentDay = 0
  → cardsRevealed = 0
  → 重置氛围系统
  → TitleScreen.show()
```

### 8.4 输入处理

- **拖拽阈值**: 8 像素 (超过则视为拖拽而非点击)
- **相机拖拽**: 按住并拖动可平移视角 (在 "ready" 状态下)
- **点击判定**: mouseUp 时如果位移 < 阈值，执行点击逻辑
- **移动端**: 支持触摸 (TouchBegin/TouchMove/TouchEnd 映射到 MouseDown/MouseMove/MouseUp)

## 20. Tween 动画引擎

### 20.1 API

```
Tween.to(target, toProps, duration, opts)
  - target: 目标表 (属性会被直接修改)
  - toProps: 目标属性值
  - duration: 持续时间 (秒)
  - opts:
    - easing: 缓动函数
    - delay: 延迟开始 (秒)
    - tag: 标签 (用于批量取消)
    - onComplete: 完成回调
    - onUpdate: 每帧回调

Tween.cancelTarget(target)  — 取消目标的所有动画
Tween.cancelTag(tag)        — 取消指定标签的所有动画
Tween.cancelAll()           — 取消所有动画
Tween.update(dt)            — 每帧驱动
Tween.damp(current, target, speed, dt)       — 阻尼插值
Tween.dampAngle(current, target, speed, dt)  — 角度阻尼插值
```

### 20.2 缓动函数 (15 种)

|------|------|
| `linear` | 线性 |

| `easeInCubic` / `easeOutCubic` / `easeInOutCubic` | 三次 |
| `easeOutQuart` | 四次 |
