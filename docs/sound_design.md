# 暗面都市 — 音效设计文档

> **版本**: v1.2
> **最后更新**: 2026-05-01

---

## 1. 设计理念

### 1.1 总体风格

**"日常生活感 + 暗涌"** — 表面是普通都市日常的声音质感，底层暗藏不安的电子脉冲。

| 维度 | 描述 |
|------|------|
| 基调 | 现代都市日常，不刻意恐怖 |
| 质感 | 风格化电子/合成音效（synth-based） |
| 暗面 | 通过低频嗡鸣、失谐、信号干扰等手法暗示异常 |
| 参考 | Persona 系列的时尚感 + 《都市传说》类的日常恐怖 |

### 1.2 设计原则

1. **克制 > 过度** — 恐怖感通过"缺失"和"异常"暗示，而非直接的尖叫/血腥音效
2. **反馈优先** — 每个玩家操作都需要即时声音反馈，无声交互 = 坏体验
3. **层次分明** — SFX、环境音、BGM 三层互不干扰，各有频段空间
4. **循环无缝** — 环境音和天气音效使用无缝循环（loop），避免接缝感
5. **两拍节奏** — 复合动作（如移动→翻卡→事件）压缩为两拍反馈，避免音效堆叠（见 §1.3）

### 1.3 "两拍"音效节奏（移动→翻卡→事件）

玩家点击棋盘格子时，实际触发一串连锁动作：角色跳跃移动 → 到达后翻开卡牌 → 弹出事件。如果每步都有独立音效（token_jump + card_flip + popup_open + evt_xxx），短时间内 3~4 个音效堆叠，体验混乱。

**设计决策：压缩为"两拍"**

| 拍 | 时机 | 音效 | 作用 |
|---|------|------|------|
| 第一拍 | 角色起跳 | `token_jump` | 操作确认 — "我点了，角色动了" |
| 第二拍 | 卡牌揭示 | `evt_xxx`（按类型选择） | 结果反馈 — "翻出了什么" |

**被移除的音效**：
- `card_flip` — 不再在移动触发的翻卡流程中播放（仍保留文件，可用于其他翻牌场景）
- `popup_open` — 不再在卡牌事件弹窗中播放；仅保留给**非移动触发**的弹窗（如裂隙确认、手动打开面板等）

**为什么落地没有音效？**
角色落地到翻卡之间间隔极短（~0.2s），加入落地音会与第二拍的事件音效冲突。token_jump 本身包含"弹起+落下"的完整声音反馈，已覆盖落地感。

## 2. 文件结构与命名规范

```
assets/audio/
├── bgm_day_light.ogg         # BGM：日间明亮相
├── bgm_day_dark.ogg          # BGM：日间暗涌相
├── bgm_dark_world.ogg        # BGM：暗面世界
├── bgm_victory.ogg           # BGM：胜利结局
├── bgm_defeat.ogg            # BGM：失败结局
│
└── sfx/                      # 全部音效
    ├── card_*.ogg             # 卡牌系统
    ├── evt_*.ogg              # 事件触发
    ├── camera_*.ogg           # 相机系统
    ├── shop_*.ogg             # 商店
    ├── weather_*.ogg          # 天气
    ├── ghost_*.ogg            # 幽灵/怪物
    ├── rift_*.ogg             # 裂隙
    └── ...                    # 其他
```

### 命名规则

- 前缀表示所属系统：`card_`、`evt_`、`camera_`、`shop_` 等
- 全部小写，单词间用 `_` 分隔
- BGM 统一以 `bgm_` 开头
- 格式统一为 `.ogg`（Vorbis）

## 4. BGM 清单

### 4.1 总览

| 文件 | 曲名 | 风格 | 时长 | 使用场景 |
|------|------|------|------|----------|
| `bgm_day_light.ogg` | Daylight Stroll | Lo-fi electronic, 温暖明亮 | ~2.4min | 日间探索 — 前半段/安全区域 |
| `bgm_day_dark.ogg` | Shadows Creeping | Dark ambient, glitch 元素 | ~3.5min | 日间探索 — 暗涌显现/危险区域 |
| `bgm_dark_world.ogg` | The Other Side | Dark synthwave, 脉冲低音 | ~3.0min | 暗面世界探索全程 |
| `bgm_victory.ogg` | Dawn Breaks | Uplifting synth, 希望感 | ~2.8min | 胜利结局画面 |
| `bgm_defeat.ogg` | Fading Consciousness | Melancholic piano + drone | ~5.5min | 失败结局画面 |

### 4.2 BGM 切换逻辑

```
游戏开始
  │
  ├─ 日间探索
  │   ├─ 暗涌值 < 阈值 → bgm_day_light（循环）
  │   └─ 暗涌值 >= 阈值 → bgm_day_dark（淡入淡出切换）
  │
  ├─ 进入暗面世界
  │   └─ bgm_dark_world（循环，叠加 dark_ambient 环境音）
  │
  ├─ 返回现实世界
  │   └─ 恢复 bgm_day_light 或 bgm_day_dark
  │
  └─ 游戏结束
      ├─ 胜利 → bgm_victory（播放一次）
      └─ 失败 → bgm_defeat（播放一次）
```

**切换方式**: 所有 BGM 切换使用 **1.5~2 秒的交叉淡入淡出**（crossfade），避免突兀中断。

## 5. 音量层级

建议的默认音量配比（可在设置中调整）：

| 层级 | 内容 | 默认音量 | 说明 |
|------|------|----------|------|
| **Layer 1** | BGM | 0.3~0.4 | 背景音乐，最低层，不抢注意力 |
| **Layer 2** | 环境音 | 0.3~0.5 | 天气、暗面环境音，营造氛围 |
| **Layer 3** | SFX | 0.6~0.8 | 操作反馈、事件音效，最高优先级 |

## 6. 集成接口参考

### 6.1 音效播放

```lua
-- 一次性音效
local sound = cache:GetResource("Sound", "audio/sfx/card_deal.ogg")
sound.looped = false
local source = scene_:CreateComponent("SoundSource")
source:Play(sound)

-- 循环环境音
local ambient = cache:GetResource("Sound", "audio/sfx/dark_ambient.ogg")
ambient.looped = true
local ambientSource = scene_:CreateComponent("SoundSource")
ambientSource:Play(ambient)
ambientSource.gain = 0.4
```

### 6.2 SFX 防机械感随机化

AudioManager 内置微随机机制，每次播放 SFX 时自动添加：
- **音调**: ±4%（`frequency * (0.96 ~ 1.04)`）
- **音量**: ±5%（`gain * (0.95 ~ 1.05)`）

连续触发相同音效时不会产生完全一致的机械感，无需额外配置。

### 6.3 BGM 播放与切换

```lua
-- BGM 通常使用 SoundSource 类型 MUSIC，便于统一控制音量
local bgm = cache:GetResource("Sound", "audio/bgm_day_light.ogg")
bgm.looped = true
local musicSource = scene_:CreateComponent("SoundSource")
musicSource.soundType = SOUND_MUSIC
musicSource:Play(bgm)
musicSource.gain = 0.35
```
